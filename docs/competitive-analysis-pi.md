# 竞品分析：pi.inc vs pi.deepvinci.tech vs md2ppt

> 研究日期：2026-05
> 目的：了解商业 AI PPT 产品的差异，找出可借鉴的设计思路

---

## 产品概览

### pi.inc（国际版）
- 定位：企业级 AI 演示平台
- 核心卖点：**多智能体设计引擎**，10–15 秒生成完整 deck
- 架构：内部多个 Agent 同时处理内容结构 + 视觉设计，不是套模板
- 数据来源：可接入企业私有知识库（产品手册、竞品信息等）
- 支持：PPTX / PDF / 长图导出；MCP API；免费无水印

### pi.deepvinci.tech（DeepVinci，国内）
- 定位：中国市场"AI-native Canva"，类 Gamma
- 核心卖点：**8 秒生成**，DeepSeek 70B 多模态模型，富动画输出
- 架构：文档级 AI / 卡片级 AI / 文字级 AI 三层编辑
- 支持：PPTX / PDF / **H5 互动网页** / 长图导出；多人协作；查看数据分析

---

## 与 md2ppt 的根本差异

| 维度 | pi.inc / DeepVinci | md2ppt（你的项目）|
|------|-------------------|-----------------|
| **定位** | 面向普通用户的 SaaS 产品 | 面向自己的开发者工作流 |
| **输入方式** | 一句话 / 上传文档 / 粘贴 URL | 自己写 Markdown 文件 |
| **内容生成** | AI 自动生成大纲 + 内容 | 你写内容，AI 只做设计+排版 |
| **设计控制** | AI 自动选主题（100+）| 你主动选 design-profile，精确控制 hex |
| **输出质量** | 视觉丰富、动画、图表 | 设计语言更精准、字体/间距可控 |
| **可编辑性** | PPTX 可编辑 | PPTX 可编辑（pptxgenjs 原生）|
| **中文支持** | 原生中文（DeepVinci 优化好）| 需手动替换字体 |
| **协作** | 多人实时协作 | 单人 |
| **费用** | 免费有限额 / $9.9–16/月 | 完全免费（自用 Claude API）|

**本质区别：**
它们是把 AI 做成了**产品**，你是把 AI 做成了**工具链**。  
它们牺牲了控制权换取速度，你保留了控制权换取精确性。

---

## 它们有哪些值得借鉴的地方

### ★★★ 高价值借鉴

**1. 多种输入方式 → 不只从零写 MD**

两款产品都支持：上传已有 Word/PDF/PPTX 文件、粘贴 URL 自动抓取内容。

**可以做的：**
- 已有文档 → 让 Claude 提炼成 deck.md 骨架，再生成 PPTX
- 粘贴文章 URL → 让 Claude 抓取正文内容 → 生成 deck.md → 走现有流程

**2. AI 先生成大纲，确认后再生成内容**

pi.inc 在生成前让用户看到并确认大纲结构，不是直接输出 30 页幻灯片。

**可以做的：**  
在 Prompt 模板里加一步：
```
Step 1: 先给我生成大纲（页数 + 每页主标题），等我确认
Step 2: 我确认后，按设计档案生成完整 PPTX
```

**3. 长图导出（Long Image）**

两款都支持把整个 PPT 导出为一张竖版长图，便于微信/社交媒体分享。

**可以做的：**
- pptxgenjs 生成 PPTX 后，用 LibreOffice 或 python-pptx 把每页转为 PNG，再纵向拼接成长图
- 一条 bash 命令搞定，适合分享给不打开 PPT 的人

### ★★ 中价值借鉴

**4. 查看数据分析（DeepVinci 特有）**

DeepVinci 跟踪每个分享链接的访问次数和每页阅读时长。

**可以做的（如果需要分享给他人）：**
- 把 PPTX 上传到 Google Slides，分享链接，用 Google Workspace 自带分析查看
- 或者导出 HTML，部署到简单服务器，用 Plausible 等轻量统计

**5. 主题一键切换**

pi.inc 的 100+ 主题可以 1 秒切换，对比同一内容不同风格。

**可以做的：**
- 在 prompt 里写"先用 guizang-monocle 生成，再用 swiss-klein-blue 生成同一内容"
- 两份 PPTX 并排比较，选更合适的

**6. H5 互动网页导出（DeepVinci 特有）**

有些场景下在线查看比 PPTX 更方便（移动端浏览、不安装软件）。

**可以做的：**
- Marp 本来就可以导出 HTML（`npm run export:html`），这条现有流程已经覆盖
- 部署到 GitHub Pages 即可分享

### ★ 低优先级借鉴

**7. 多人协作**
- 你是个人用户，暂时不需要

**8. 企业知识库接入（pi.inc）**
- 如果有大量个人笔记、文档要引用，可以考虑用 RAG 接入 Claude
- 暂时手动粘贴内容更简单

---

## 总结：你的项目真正的优势

商业产品解决的是"从 0 快速生成"的问题，但有几个死穴：

1. **设计控制权不足**：主题由 AI 随机选，颜色不可精确控制
2. **内容质量不稳定**：AI 可能编造内容、逻辑混乱
3. **中文字体问题**：国际产品普遍存在
4. **付费门槛**：真正好用的功能都在付费版

**你的项目的真正价值**：
- **你控制内容**（deck.md 你写），不依赖 AI 编造
- **设计语言精确**（hex 锁死，字体锁定，版式铁律）
- **完全免费**（自己的 Claude/Codex，自己的 API）
- **工作流可复用**（设计档案可以持续积累）

**适合加进来的最小改动**：
1. 在 Prompt 里加"先生成大纲供确认"步骤
2. 支持从 URL/文档提炼 deck.md 骨架的快速命令

这两个改动可以让你的工作流在保持精确控制的同时，大幅降低"从零写 MD"的摩擦。
